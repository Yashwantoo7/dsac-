                                                                                                  young_cooper taboverspace 2 months ago

                                                                                                  when we do slow == fast, this only tells that there exist a loop in LL, not the position that where is loop.
                                                                                                  so after traversing the LL, and stopping where fast== slow(means loop exists).
                                                                                                  There may a chance that loop starts from head so we need to reach there. In this case, after traversing(I have take many cases, got same result) both slow and fast pointers will be at head, so there is a check whether slow == head which means loops starts from head. To remove this loop we iterate with slow pointer till slow->next = head and then we do slow->next = NULL
